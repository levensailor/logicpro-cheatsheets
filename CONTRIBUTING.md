# Contributing

## Branching Workflow
1. Pull the latest `main` branch.
2. Create a new feature branch:
   - `git checkout -b feature/<short-description>`
3. Make your changes and commit with clear messages.
4. Push your feature branch to origin.

## Pull Request Requirements
When opening a pull request, include:
- What changed.
- Why this change should be added to the repository.
- How to test and validate the behavior.
- Any screenshots for UI updates.

## Quality Checklist
- Keep cheat sheets editable via component/data updates.
- Avoid turning any full sheet into a static page image.
- Keep embedded visuals limited to section-level assets only.
- Update `CHANGELOG.md` when adding new features.
